## 🖥️ Interfaces de usuário e Experiência (UX)

> O segredo é mostrar que você entende que a plataforma oferece interfaces diferentes para perfis de usuários diferentes.  

O ServiceNow oferece múltiplas formas de interação, adaptadas às necessidades de diferentes perfis, desde usuários finais até administradores de sistema.  

### 1. Interface Clássica (Core UI)
É a interface principal para administradores e "power users".  
Utiliza a Navegação Unificada (Next Experience) para organizar tabelas, formulários e listas.  
- Foco: Configuração, desenvolvimento e gestão técnica.  
- Componentes: Banner superior, menu de navegação (All, Favorites, History) e área de conteúdo principal.  

### 2. Service Portal
Uma interface web amigável, configurável e responsiva, destinada ao usuário final.  
- Foco: Autoatendimento (Self-Service).  
- Uso: Solicitação de itens do Service Catalog, consulta à Knowledge Base (Base de Conhecimento) e acompanhamento de chamados.  
- Tecnologia: Baseado em widgets, permitindo customização total com HTML, CSS e AngularJS.  

### 3. Workspaces (Espaços de Trabalho)
Interfaces modernas de "painel único" (Single Pane of Glass) projetadas para alta produtividade.  
- Foco: Agentes de atendimento e analistas.  
- Vantagem: Permite que o agente veja o contexto completo de um chamado, interaja com o usuário via chat e consulte a base de dados sem trocar de aba.
  Exemplo: Service Operations Workspace.

### 4. ServiceNow Mobile
Acesso à plataforma através de aplicativos nativos para dispositivos móveis (iOS e Android).  
- Foco: Mobilidade para técnicos de campo e aprovações rápidas para gestores.  

### 💡 Por que isso é relevante para um projeto?
Ao documentar as interfaces, demonstramos o domínio sobre a experiência do usuário (UX).
