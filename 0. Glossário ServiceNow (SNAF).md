## 📖 Glossário ServiceNow (SNAF)

Este glossário contém os termos essenciais e as suas definições para apoiar a jornada na ServiceNow University.

## A

* **Access Control (Controle de Acesso):** Regra de segurança individual que define quem pode acessar registros ou campos.
* **Access Control List (ACL):** Conjunto de regras de segurança utilizadas para controlar permissões de criar, ler, atualizar e excluir (CRUD).
* **Application Platform-as-a-Service (aPaaS):** Serviço em nuvem que fornece ferramentas e infraestrutura para desenvolver, implementar e gerir aplicações personalizadas.
* **Application Engine Studio (AES):** Ferramenta intuitiva que permite a criação e implementação rápida de aplicações na plataforma ServiceNow.
* **Application Scope (Escopo da Aplicação):** Define os limites dentro dos quais uma aplicação opera. Pode estar num escopo personalizado ou em Global.
* **Application Scoping (Escopo de Aplicação):** Protege as aplicações identificando e restringindo o acesso a artefactos e dados disponíveis.
* **Approver (Aprovador):** Função (role) no ServiceNow que permite aos utilizadores visualizar ou modificar registos de aprovação direcionados a eles.
* **Automated Test Framework (ATF):** Conjunto de ferramentas para automatizar testes de aplicações, garantindo qualidade e reduzindo esforço manual.

---

## B

* **Baseline Implementation (Implementação Base):** Conjunto de aplicações instaladas numa instância ServiceNow antes de qualquer configuração ou personalização.
* **Base Table (Tabela Base):** Tabela principal no topo de uma hierarquia, da qual outras tabelas podem herdar campos e comportamentos.
* **Business Rule (Regra de Negócio):** Executada antes ou depois de uma ação no banco de dados (ex: inserir, atualizar, excluir ou consultar).

---

## C

* **Capstone Project:** Projeto final que envolve a aplicação dos conhecimentos e competências adquiridos ao longo do curso.
* **Child Class (Classe Filha):** Tabela que estende outra tabela.
* **Choice Lists (Listas de Escolha):** Tipo de campo que permite ao utilizador selecionar opções predefinidas.
* **CI Class Manager:** Local central para explorar a hierarquia de classes do CMDB, definições de tabelas de Itens de Configuração (CI) e classes de CIs.
* **Client (Cliente):** Interface do utilizador ou front-end da plataforma ServiceNow onde os utilizadores interagem com o sistema.
* **Client Script:** Scripts criados para realizar ações no lado do cliente, como atualizar campos automaticamente ou exibir avisos.
* **CMDB (Configuration Management Database):** Série de tabelas e campos que contêm todos os Itens de Configuração (CIs) controlados pela empresa.
* **Common Service Data Model (CSDM):** Estrutura baseada no CMDB que identifica onde colocar dados para os produtos utilizados.
* **Configuration (Configuração):** Ajuste de funcionalidades nativas, como listas de categorias, vistas de lista, catálogos de serviços e campos base.
* **Configuration Items (CIs):** Dispositivos ou aplicações tangíveis ou intangíveis registados no CMDB.
* **Context Menus (Menus de Contexto):** Fornecem diferentes níveis de controlo para uma determinada vista de lista.
* **Coalesce Fields (Campos de Coalescência):** Utilizados como chave exclusiva durante a importação de dados para evitar duplicados.
* **Core Table (Tabela Principal):** Tabelas que existem nativamente no sistema base.
* **Customization (Personalização):** Desenvolvimento de novas funcionalidades, como alterar widgets do portal ou criar tabelas para requisitos específicos.
* **Customer Actions:** Etapas que os clientes podem implementar e visualizar no Security Center (ex: pendentes, em curso ou concluídas).

---

## D

* **Data Pill:** Contentor de dados utilizado em ações subsequentes dentro de um fluxo de trabalho.
* **Data Policy (Política de Dados):** Regra que impõe consistência dos dados (ex: campos obrigatórios ou apenas leitura) no lado do servidor.
* **Dashboards (Painéis):** Recurso que permite visualizar múltiplas análises de dados e métricas numa única interface partilhável.
* **Dependency View (Vista de Dependência):** Interface gráfica interativa para visualizar relações entre Itens de Configuração (CIs).
* **Discovery:** Ferramenta que varre a rede para inventariar dispositivos e aplicações, atualizando automaticamente o CMDB.
* **Dot-Walking:** Método para obter informações de tabelas relacionadas através de campos de referência.

---

## E

* **Encryption at Rest (Criptografia em Repouso):** Proteção de dados armazenados no disco através de criptografia.
* **Encryption in Transit (Criptografia em Trânsito):** Segurança no acesso às instâncias via HTTPS e métodos de transferência de dados (APIs, FTPS, etc.).

---

## F

* **Field (Campo):** Coluna individual de dados numa tabela.
* **Field Attributes (Atributos de Campo):** Valores e propriedades dos dados inseridos num campo.
* **Field Label (Rótulo do Campo):** Nome amigável que o utilizador vê na interface.
* **Field Name (Nome do Campo):** Nome técnico único que o sistema utiliza para identificar o campo.
* **Filters (Filtros):** Condições aplicadas a uma lista para isolar subconjuntos de dados.
* **Flow (Fluxo):** Sequência de operações executadas pelo sistema (ex: atualizar um registo ou solicitar aprovação).
* **Form Builder:** Ferramenta para configurar formulários adicionando, removendo ou reordenando campos.

---

## G

* **Groups [sys_user_group]:** Tabela que armazena grupos de utilizadores com um propósito comum (ex: equipas de suporte).

---

## H

* **Hardening Score:** Pontuação visualizada no Security Center que indica o nível de conformidade com definições de segurança.
* **HR Service Management:** Suíte de aplicações para gerir serviços de RH (onboarding, gestão de casos, etc.).

---

## I

* **Impersonation (Personificação):** Funcionalidade que permite aos administradores agir como outro utilizador para testes e suporte.
* **Import Set:** Ferramenta para importar dados de fontes externas e mapeá-los para tabelas do ServiceNow.
* **Incident [incident]:** Tabela utilizada para gerir interrupções não planeadas ou reduções na qualidade dos serviços de TI.
* **Instance (Instância):** Cópia isolada do ambiente ServiceNow específica para um cliente.
* **ITIL:** Conjunto de boas práticas para a entrega de serviços de TI de alta qualidade.
* **ITSM (IT Service Management):** Conjunto de aplicações para gerir serviços de TI (Incidentes, Mudanças, Problemas).

---

## K

* **Key Management Framework (KMF):** Interface segura para gestão de chaves criptográficas na instância.
* **Knowledge Base (Base de Conhecimento):** Repositório de informações onde se pode criar, rever e consultar artigos úteis.

---

## L

* **List Collector:** Interface utilizada para selecionar e mover múltiplos itens entre listas.
* **List Editor:** Permite editar valores de campos diretamente numa lista sem abrir o registo.
* **List Views:** Configurações de colunas e campos específicos para diferentes atividades de trabalho.
* **Logging and Monitoring:** Capacidades para configurar e encaminhar logs para sistemas externos (SIEM).

---

## M

* **MID Server:** Aplicação leve que comunica entre a instância ServiceNow e sistemas externos na rede local.

---

## N

* **Natural Language Query (NLQ):** Permite pesquisar dados utilizando linguagem comum em vez de código ou filtros complexos.
* **Now Assist:** Ferramenta de IA generativa que ajuda a automatizar fluxos de trabalho e resumir informações.
* **Notifications (Notificações):** Alertas disparados por eventos (ex: e-mails enviados quando um incidente é resolvido).

---

## P

* **Parent Class (Classe Pai):** Tabela de origem a partir da qual outra tabela é estendida.
* **Performance Analytics:** Ferramenta avançada para acompanhar tendências e analisar indicadores chave de desempenho (KPIs).
* **Plugins:** Componentes modulares que ativam funcionalidades ou aplicações adicionais na instância.

---

## R

* **Record (Registo):** Corresponde a uma linha numa tabela da base de dados.
* **Record Producers:** Formulários simplificados para utilizadores criarem registos de tarefas através do portal.
* **Role (Função/Papel):** Define permissões de acesso a aplicações, módulos e dados.

---

## S

* **Schema Map (Mapa de Esquema):** Representação gráfica das relações entre tabelas.
* **Service Catalog (Catálogo de Serviços):** Loja de serviços e produtos disponíveis para pedido pelos utilizadores.
* **Service Mapping:** Mapeia relacionamentos entre CIs para definir serviços de negócio completos.
* **System Administrator:** Função com acesso total a todas as configurações e dados da plataforma.

---

## T

* **Table (Tabela):** Coleção de registos na base de dados.

---

## U

* **Update Set:** Grupo de alterações de configuração que podem ser movidas entre instâncias.
* **UI Policy:** Regra que altera dinamicamente o comportamento do formulário (ex: tornar campo obrigatório conforme a escolha do utilizador).

---

## V

* **Virtual Agent:** Assistente virtual (chatbot) que ajuda os utilizadores com tarefas e dúvidas comuns.

---

## W

* **Workflow Studio:** Ambiente unificado para criar e gerir fluxos de trabalho, automações e lógica de negócio.
